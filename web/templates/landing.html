stage

An observability record for ensemble runs.

When you run a scenario with Stage configured, every event streams as it happens -- actor messages, tool calls, state changes at each step, cost accumulations, and grader output. The run URL appears in your terminal before the scenario starts. You can open it in a browser and watch the trace build in real time, or come back later and step through each event as if the run were still live. A single run tells you not just what the outcome was but why: which tool call consumed the budget, where the agent's reasoning diverged, which predicate the grader caught.

Sweeps group runs that vary a parameter -- backend, seed, world -- so you can compare model behavior across conditions in one place. Training runs record the job alongside the evaluations that preceded it, so the record of what changed and whether it worked stays connected rather than scattered across logs.
